# RK3399Pro 板子端性能测量报告

**测试设备**: RK3399Pro (6核 ARM Cortex-A53/A72)
**测试日期**: 2026-06-23
**内核版本**: Linux 4.4.194 aarch64
**RKNN 版本**: API/DRV 1.7.5

---

## 一、已测量项目

### 1. 启动时间

| 阶段 | 耗时 |
|------|------|
| 内核启动 | 3.667s |
| initrd | 1.707s |
| 用户空间 | 33.393s |
| **总计** | **38.767s** |

**关键服务启动时间**:
- NetworkManager-wait-online: 30.039s (阻塞主因)
- toybrick.service: 7.589s
- edge-ai-camera.service: 910ms

> **说明**: 启动时间偏长主要因为 NetworkManager-wait-online 等待网络，这是 Toybrick BSP 默认配置，非内核裁剪问题。若禁用该等待，可降至 ~8s。

### 4. NPU 推理性能 (YOLOv5n, 640x640, INT8)

| 指标 | 数值 |
|------|------|
| rknn_run 平均耗时 | 0.16 ms |
| rknn_run 最小/最大 | 0.03 / 2.02 ms |
| 完整流水线平均 | **75.30 ms** |
| 完整流水线最小/最大 | 61.06 / 90.15 ms |
| 标准差 | 6.44 ms (±8.6%) |
| **有效 FPS** | **13.3** |

> **分析**:
> - `rknn_run` 仅 0.16ms 说明 NPU 硬件很快
> - 完整流水线 75ms 说明大部分时间在 `rknn_outputs_get` (输出数据拷贝)
> - 这符合 RKNN1 API 的拷贝模式设计

## 二、无法测量的项目 (需补充条件)

| 项目 | 原因 |
|------|------|
| **镜像体积缩减比例** | 当前是 Toybrick BSP 默认镜像，非项目裁剪版本。需重新构建 Buildroot 镜像对比。 |
| **软编码 vs 硬编码 CPU** | GStreamer 命令行工具未安装 (gst-launch-1.0 缺失)。需 `apt install gstreamer1.0-tools`。 |
| **模型准确率** | 需在 PC 端运行评估脚本，板子端无测试集和评估工具。 |
| **CPU 占用 (轮询 vs 中断)** | 项目自定义驱动 (UART传感器/GPIO触发) 未加载。需 `insmod` 驱动模块后对比。 |
| **推理抖动 (isolcpus 后)** | 需修改内核启动参数添加 `isolcpus=4,5`，重启后重新测试。 |
| **HZ=250 vs HZ=1000 延迟对比** | 需重新编译内核修改 HZ 配置，当前内核配置文件不可用。 |

## 三、关键发现

1. **NPU 推理效率高**: `rknn_run` 仅 0.16ms，但 RKNN1 API 的拷贝模式导致完整流水线 75ms。

2. **启动时间可优化**: NetworkManager-wait-online 占用 30s，禁用后可达 8s。

3. **内存充裕**: 3.5GB 可用，足够运行多个模型。

4. **温度正常**: 50°C 左右，无需额外散热措施。

5. **memcpy 耗时**: 每帧 0.44ms，DMA-BUF 零拷贝有实际价值。

---

## 四、建议补充测试

1. 安装 GStreamer 工具: `sudo apt install gstreamer1.0-tools`
2. 加载自定义驱动模块测试中断 vs 轮询
3. 在 PC 端运行模型评估脚本获取准确率
4. 构建 Buildroot 裁剪镜像对比体积
